## 1. 原住民族委員會開放資料平台

- **網址**：https://data.cip.gov.tw/
- **主管機構**：原住民族委員會（Council of Indigenous Peoples）
- **資料性質**：以原住民族為主體的行政統計與政策資料
- **涵蓋主題**：△ 人口統計、就業、教育、衛生福利、土地、經濟等（依原民會業務分類）
- **資料單位**：△ 行政區（縣市/鄉鎮市區）為主，部分含族別分類
- **資料格式**：△ CSV、JSON（依政府開放資料標準）
- **存取方式**：△ 直接下載、部分提供 API
- **更新頻率**：？ 依資料集而異，部分為年度更新
- **授權條件**：✓ **原住民族委員會開放資料使用規範 1.0 版**（非通用「政府資料開放授權條款」，為 CIP 獨立制定）——含原住民族權益尊重條款、傳統智慧創作專用權保留條款、智慧創作未來追溯條款
- **詮釋資料品質**：？ 預期有資料集名稱、描述、關鍵字、提供機關、更新頻率等欄位
- **資料集數量**：✓ **253 筆**資料集
- **分類體系**：✓ 24 個分類——民族人口、原民鄉鎮、原民法規、原民部落、歲時祭儀放假日、文學、音樂、藝術、學校課程、學生社團、圖書資訊、語言教育、就學狀況、社團、公司行號、通訊錄、社會補助、職業勞動、就業狀況、經濟狀況、會計月報、決算、預算、其他
- **API 端點**：✓ REST API 存在，格式為 `https://data.cip.gov.tw/API/v1/dump/datastore/{資料集ID}`，每筆資料集提供 JSON / CSV / XML 三種格式
- **資料集 ID 格式**：✓ `A53000000A-{funno}-{format_code}`（如 `A53000000A-114025-001` 為 JSON）

### 盤點備註
- **平台架構為 ASP.NET（非 CKAN）**：首頁導向 `/Home/Default.aspx`，資料查詢頁為 `/Home/DataMenu.aspx`（依主題分類）和 `/Home/UnitMenu.aspx`（依機關分類）。先前以 CKAN 路徑（/dataset）存取時回傳錯誤，是路徑不存在而非伺服器異常。
- **API 可直接下載資料**：每筆資料集提供 XML、CSV、JSON 三種下載連結，無需認證，為五個來源中存取門檻最低的
- **授權條款為獨立制定（重要更正）**：2026-04-12 實測 `/Home/Copyright.aspx` 原文確認，CIP 使用的是自行制定的「原住民族委員會開放資料使用規範 1.0 版」，**並非通用政府資料開放授權條款**。該規範明確要求：
  - 利用者「尊重原住民族權益，不妨礙或侵害原住民族之生活方式、傳統習俗、服飾、文化及價值觀、社會經濟組織型態、資源利用方式、土地利用與管理模式等」
  - 原住民族傳統智慧創作專用權被列為授權不及於的保留範圍
  - 若未來智慧創作專用權人主張權利，利用者同意不得對原民會或資料提供人請求賠償
- 253 筆資料集中涵蓋面廣（人口、就業、教育、土地、經濟、語言、文化、部落 GIS 點位等），但資料量分布不均——就業與經濟相關資料集佔大宗
- 值得注意的資料集（2026-04-12 實測確認）：
  - `原住民族部落位置GIS點位表`（funno=111027）：739 筆，含 X/Y 座標與 WGS84 經緯度、行政區、族別、部落族語羅馬拼音
  - `核定部落結果`（funno=112041）：746 筆，含核定民國年份、族語名稱、族名、行政區（無座標，與 111027 互補）
  - `55個原住民族鄉鎮市區`（funno=112039）：55 筆官方認定鄉鎮清單，含各鄉鎮公所網址
- 核心限制：大多數資料以行政區為單位，部落層級資料僅存在於少數主題型清冊

## 2. 政府資料開放平臺 (data.gov.tw)

- **網址**：https://data.gov.tw/
- **主管機構**：國家發展委員會（National Development Council）
- **資料性質**：跨部會政府開放資料彙整平台
- **涵蓋主題**：✓ 涵蓋所有政府機關開放資料，可搜尋「原住民」相關資料集
- **資料單位**：✓ 依資料集而異，多數為行政區層級
- **資料格式**：✓ CSV 為主，部分提供 JSON、XML
- **存取方式**：✓ 直接下載（透過資料提供機關的 URL）、部分有 API
- **更新頻率**：△ 依資料集而異，詮釋資料中有「更新頻率」欄位但不一定填寫
- **授權條件**：✓ 政府資料開放授權條款（參見 data.gov.tw/license）
- **詮釋資料品質**：✓ 每筆資料集具備：編號、標題、關鍵字、提供機關、格式、存取 URL；使用 Schema.org 結構化標記
- **原住民族相關資料集數量**：✓ 透過全平台 CSV 匯出（`/api/v2/rest/dataset/export`）篩選，含「原住民」關鍵字的資料集共 **833 筆**（其中標題含「原住民」539 筆，僅描述或其他欄位含「原住民」294 筆）
- **提供機關分布**：✓ 原住民族委員會 240 筆（最大宗）、臺南市政府社會局 32 筆、統計處 25 筆、臺南市政府民政局 24 筆、臺中市原民會 21 筆、臺北市原民會 19 筆、行政院主計總處 16 筆、臺中市教育局 14 筆等，橫跨中央與地方數十個機關
- **平台總量**：✓ 全平台共 52,914 筆資料集（2026-04-12），CSV 匯出為 74,041 行（含多筆資源的展開列）
- **API 詮釋資料欄位**：✓ 每筆資料集含 50+ 欄位，包括：agency_name、agency_tid、title、content（描述）、license_name、category_name、topic_name、updatefreq_desc、all_file_format_name、dataset_resource_description、quality_badge_type 等
- **機關分類**：✓ API 回傳 aggregations 含 agency_tid 與 topic_tid 分類，但未見「原住民族」專屬分類

### 盤點備註
- 此平台為彙整入口，實際資料由各機關維運；品質與更新狀態取決於原始提供者
- 原住民族相關資料散布於多個提供機關（原民會、內政部、教育部、衛福部等）
- **API 可用性問題**：公開 API 的搜尋功能實質上不運作——query 參數被忽略，agency_tid 篩選未生效。前端搜尋可能使用不同的內部端點。這意味著程式化盤點原住民族相關資料集的數量與範圍目前不可行。
- 核心限制：搜尋「原住民」可得結果（僅透過前端），但無獨立分類體系；同一主題可能重複出現於不同機關的資料集中
- 與 data.cip.gov.tw 存在資料重疊，但缺乏明確的對應關係說明

- **資料單位**：✓ **TICD 現已遷移至 OSF**（https://osf.io/esw67），以「部落」為核心資料單位
- **資料格式**：？ 預期包含 GIS 圖資、調查問卷資料、統計表格等多元格式

### 盤點備註
- **TICD 已從中研院遷移至 OSF 平台（但中研院保有正式轉址子域名）**：原網域 ianthro.tw 已無法解析，中研院人社中心首頁亦無連結。但 TICD 專案實際上仍活躍於 OSF（https://osf.io/esw67），最後更新日期為 2025-05-16。
  - **正式轉址子域名**：`https://ticd.rchss.sinica.edu.tw/`（中研院正式子域名，內容為 HTML meta refresh 自動轉址至 OSF 專案）——這是機構層面保留的官方入口，雖然主站首頁無連結，但子域名公開可達。此子域名記載於 OSF 專案的「TICD_原住民部落基礎開放資料_正式網址.txt」檔案中。
  - 實質架構是「機構保留正式轉址 URL + 內容由個人主導的 OSF 專案承擔」的折衷——不是完全的機構脫落
- **TICD OSF 專案詳情**：
  - 全名：TICD: Taiwan Indigenous Communities Open Data 台灣原住民族部落開放資料庫
  - 主持人：林季平（Ji-Ping Lin），中央研究院
  - 授權：GNU General Public License (GPL) 3.0
  - DOI：10.17605/OSF.IO/ESW67
  - 來源計畫：原住民族委員會委託「當代台灣原住民族研究」(2013-2022)，另獲國科會多項補助
  - 子專案：
    - **iqTICD**：綜合查詢系統（Integrated Query System）
    - **TIMD**：台灣原住民各族群遷徙動態（Migration Dynamics）
    - **TIHV**：高解析度視覺化圖形資料庫（人口分布、遷徙、傳統部落）
  - 最新發表：2025 年 Springer Nature Social Sciences 論文，探討難以觸及人口的遷徙動態

- 中研院的資料庫生態高度分散，TICD 遷移至 OSF 這件事本身就說明了機構維運的不穩定性——一個重要的學術資料庫從機構域名消失，卻在個人主導的開放平台上延續
- 核心限制：雖然 OSF 是開放平台，但 TICD 的發現門檻仍然很高——從中研院網站完全無法導航至此；需要事先知道 OSF 連結才能找到

## 4. 原住民族語言資料庫 (AILT)

- **網址**：https://ailt.ilrdf.org.tw/
- **主管機構**：財團法人原住民族語言研究發展基金會（ILRDF，維運單位）；**著作權持有者為原住民族委員會**（footer 聲明「Copyright © 2022 Council of Indigenous Peoples」）
- **設立法源**：依《原住民族語言發展法》第 9 條「需建置原住民族語言資料庫，積極保存原住民族語料」；2017 CIP 委託規劃、2018 第一期建置、2022（民國 111 年）完成建置；配合 2022-2026 國家語言整體發展方案
- **資料性質**：原住民族語言語料保存與研究
- **涵蓋主題**：✓ 四大語料類型：
  - **口語語料**：族語口述影音紀錄（含生命史、部落生活等主題）
  - **書面語料**：族語書寫文本
  - **典藏資料**：未經轉寫的原始檔案（文字影像、未處理音檔）
  - **學習者語料庫**：語言學習材料
  - **平行語料**：✓ 目前僅阿美語（Amis）
- **涵蓋語言**：✓ 16 族語——阿美語、泰雅語（賽考利克）、排灣語、布農語（郡群）、卑南語、魯凱語、鄒語、賽夏語、達悟語、邵語、噶瑪蘭語、太魯閣語、撒奇萊雅語、賽德克語（德固達雅）、拉阿魯哇語、卡那卡那富語
- **資料單位**：✓ 影音層級（單篇口述紀錄）、文件層級（單篇文本）、詞彙層級（輸入法詞庫支援 42 族語詞庫）
- **資料格式**：✓ 影片（YouTube 嵌入）、文件影像（PDF 預覽）、圖片資源
- **存取方式**：✓ 線上瀏覽與搜尋（依族別/語料類型導覽）；✗ 語料頁面無下載選項；有登入系統（暗示部分功能需帳號），可用操作僅限搜尋與進階搜尋；無 API
- **更新頻率**：？ 未明確標示
- **授權條件**：✓ **全站無任何使用條款頁**（2026-04-12 實測 /terms、/tos、/copyright、/agreement 等常見路徑全部回傳 404，導覽列中也無條款/規範/著作權/授權相關連結）。唯一授權宣告為 footer「原住民族語言研究發展基金會© 版權所有 Copyright © 2022 Council of Indigenous Peoples. All Right Reserved.」——**實質為預設著作權（All Rights Reserved）而非主動授權**，且**著作權人是 CIP 而非 ILRDF**
- **詮釋資料品質**：△ 每筆資源具備標題、族群歸屬、觀看次數、機構歸屬；但缺乏細粒度語言學標註（轉寫、語者資訊等）

### 盤點備註
- 語料以「保存」為核心目標，設計上偏向展示與瀏覽，非結構化資料再利用
- 「版權所有」的授權條件意味著外部研究者無法直接引用或再製語料內容
- 平行語料僅限阿美語，其餘 15 族語缺乏跨語言對照資源
- 核心限制：語料缺乏使用情境資訊（何時、何地、為何產生），難以與人口統計或文化資料對接；著作權保留的授權模式與開放資料生態不相容
- 值得注意：42 族語輸入法詞庫為新上線功能，顯示平台仍在積極發展中
- **著作權人與治理一致性的矛盾（2026-04-12 實測發現）**：CIP 是 AILT 的著作權持有者，且 CIP 自己的「原住民族委員會開放資料使用規範 1.0 版」含原住民族權益條款與傳統智慧創作保留；**但 AILT 並未採用 CIP 自家的使用規範**，實質上把族語這類文化敏感度最高的資料置於最嚴格的預設著作權下。這是機構內部治理不一致的典型例子——同一權利人對不同資料採取截然不同的開放態度。
- **設計意識存在但未實作**：about 頁自述 AILLA、PARADISEC 等國際族語資料庫有「分級訪問權限制度」——AILT 團隊知道這類設計但未實作類似機制，這是設計缺口而非疏忽

- **主管機構**：文化部（Ministry of Culture），實際維運單位為國立臺灣歷史博物館（地址：臺南市安南區長和路一段250號）

- **涵蓋主題**：✓ 九大分類：人物與團體、藝術與人文、社會與政治、民俗與宗教、產業與經濟、空間/地域與遷徙、**族群與語言**、生物/生態與環境、其他
- **資料單位**：✓ 單筆典藏素材（圖片、影片、聲音、文件、3D）
- **資料格式**：✓ 五種素材類型：圖片、影片、聲音、文件、3D
- **存取方式**：✓ 線上瀏覽與搜尋（支援分類篩選與關鍵字搜尋）；✓ **OpenAPI 存在**，需申請 API Key
- **OpenAPI 詳情**：
  - 端點：`https://tcmbdata.culture.tw/opendata/openapi`（GET）
  - 文件：Swagger UI 位於 `https://tcmbdata.culture.tw/swagger-ui/`

  - 回傳格式：JSON
  - 回傳欄位：identifier、title、description、imageLic（圖片授權）、contentLic（內容授權）、images 陣列、repImage、subjects、keywords、formats、creator、contributors（含角色/姓名）、dates（含類型/起訖日期）、places（含座標/縣市/鄉鎮）、publisher、language、storage、size、URL

### 盤點備註
- **OpenAPI 是五個來源中唯一正式提供程式化存取的管道**，但其申請門檻（固定 IP）對非機構使用者構成障礙
- API 回傳的 places 欄位含座標與行政區資訊，這是五個來源中唯一可能建立地理對應的結構化欄位
- 混合授權模式是本專案最需要關注的特徵：同一平台內不同物件的 imageLic 和 contentLic 可能完全不同
- 資料來源包括政府機關、地方文化單位與社群貢獻者，生產邏輯高度異質
- 核心限制：敘事型內容難以結構化分析；原住民族相關內容（~8,000 筆）混雜於整體典藏中，有「族群與語言」分類但缺乏獨立的原住民族資料治理標示

- **10 種授權標示架構（2026-04-12 實測 /zh-tw/license 原文修正）**：
  - **屬於開放資料（5 種）**：PDM（公眾領域標章）、CC0（公眾領域貢獻宣告）、OGDL（政府資料開放授權條款）、CC BY（姓名標示）、CC BY-SA（姓名標示─相同方式分享）

### 關鍵發現

1. **「部落」作為資料單位僅存在於一個來源**：TICD 是唯一以部落為核心單位的結構化資料庫，但它已從機構網站消失，遷移到個人主導的 OSF 平台。CIP 有部落 GIS 點位表（funno=111027）和核定部落清冊（funno=112041），但這些是清冊而非資料庫。
2. **存取方式的光譜從完全開放到完全封閉**：CIP 提供免認證 REST API（最開放），TCMB 需申請 API Key 加固定 IP，AILT 則連下載功能都沒有（最封閉）。data.gov.tw 的公開搜尋 API 名存實亡（query 參數被忽略，永遠回傳全部 52,914 筆）。openmuseum 則是**規模與 API 開放性反比**的極端案例——117 萬件藏品但完全無公開 API。
3. **授權條件與集體權利部分脫節——但有一個在地化例外**：五個來源涵蓋多種授權模式（CIP 獨立使用規範、政府開放授權、GPL、TCMB 10 種標示混用、AILT 預設著作權）。其中 **CIP 的使用規範含在地化的集體權利條款**（尊重原住民族權益、保留傳統智慧創作專用權），是唯一對原住民族集體權利有實質回應的來源；但其仍未對接 CARE 原則、OCAP、TK Labels 等國際框架。其餘四個來源則完全沒有集體權利相關條款。

   **一個值得特別點出的機構內部矛盾**：CIP 是 AILT 族語資料庫的著作權持有者，理論上 CIP 自己的「原住民族委員會開放資料使用規範 1.0」可以直接套用於 AILT，但 AILT 並未採用，仍處於預設「All Rights Reserved」狀態。這意味著同一個機構（CIP）對自家行政統計資料採在地化集體權利條款，卻把文化敏感度更高的族語資料置於更嚴格的全面著作權保留下——治理原則未能跨資料類型一致延伸。
4. **敘事資料與數值資料之間的鴻溝**：AILT 和 TCMB 擁有豐富的質性內容，但缺乏可被量化分析流程使用的結構化詮釋資料；反之，CIP 統計有結構但缺乏脈絡。TCMB 的 OpenAPI 在一定程度上橋接了這個鴻溝（提供結構化詮釋資料存取質性內容），但需要申請門檻。
5. **沒有任何來源採用國際原住民族資料治理標示**：TK Labels、CARE Principles、OCAP 等國際框架在這五個來源中完全沒有被實作。但 CIP 有台灣在地化的替代方案（見第 3 點）——未來是否延伸至 CARE/OCAP 等，或與國際框架對接，是待觀察的治理議題。
6. **跨來源串聯的技術基礎極為薄弱**：五個來源之間沒有共享的識別碼或詮釋資料標準。唯一的潛在串聯點是：CIP 的核定部落清冊 + TICD 的部落資料 + TCMB API 的 places 座標欄位——但這三者的命名慣例、座標系統與涵蓋範圍是否一致，仍待驗證。
7. **機構維運的脆弱性**：TICD 從 ianthro.tw 消失遷移到 OSF、data.cip.gov.tw 的 CKAN 路徑不存在（實為 ASP.NET）、data.gov.tw 的搜尋 API 失靈——這些都不是「資料不存在」，而是「資料存在但入口不穩定或不可預期」。

8. **六個來源中無一由原住民族社群、部落或自治組織持有**：依 OCAP Possession 原則的殖民脈絡分析，「國家設立的原住民族專責機關」（如原住民族委員會、原住民族語言研究發展基金會）仍屬國家機構延伸，不構成社群自持。依此標準——
   - CIP、AILT：由國家設立的原住民族專責機構保管 → 仍屬國家延伸
   - data.gov.tw、TCMB、openmuseum：由一般政府機關／博物館／學術機構保管 → 非原住民族相關
   - TICD：由中立、獨立於國家的 OSF 學術平台保管 → 這是六個來源中**唯一非國家延伸**的保管形式，但也非社群自持

   換言之，在現行六個資料源的生態中，**沒有任何管道讓資料真正回到部落／自治組織手中**。這比「沒有採用國際框架」更具體——即使 CIP 有自家在地化條款，這些條款仍是國家機關的自我立法結果，非社群參與的產物。

9. **規模與開放性反比**（2026-04-13 新增 openmuseum 後發現）：openmuseum 擁有 117 萬件藏品——規模為六來源之冠，但**完全無公開 API**。反觀最小的 CIP（253 筆）提供免認證 REST API。規模與 API 可及性之間沒有正相關，有時甚至呈反比——規模大反而意味著 schema 複雜度高、機構協作層級多、公開 API 反而難以建立與維護。這是「資料存在但不可用」的另一種型態：不是路徑失靈（如 data.gov.tw 的搜尋 API），也不是申請門檻（如 TCMB 的固定 IP），而是**根本沒有介面**。
